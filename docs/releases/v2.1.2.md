# GrtBlog v2.1.2

- 发布日期（UTC）：2026-07-14
- 对比基线：v2.1.1
- 发布通道：Stable

v2.1.2 是针对 v2.1.0 以来 ISR 静态目录权限问题的紧急修复版本。

## 修复内容

- 修复 Renderer 重建或重启后可能在共享 `storage/html` 中留下 root 所有目录，导致 Server 无法创建 `categories`、`columns`、`auth` 等 ISR 路径的问题。
- Renderer 现在与 Server 统一使用 `app:app`（UID/GID `10001:10001`）写入共享存储。
- Renderer 启动时会修正 `storage/html` 与 `storage/meta` 的所有权，再以非 root 用户复制前端资源、写入 ISR manifest 并运行 Node 服务。
- 取消会保留 root 文件元数据的归档复制，避免后续容器重建再次破坏 Server 的写权限。

## 升级说明

推荐重新运行一键安装脚本升级到 v2.1.2：

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/grtsinry43/grtblog/main/deploy/install.sh)
```

国内网络可使用：

```bash
bash <(curl -fsSL https://cnb.cool/grtsinry43/grtblog/-/git/raw/main/deploy/install.sh)
```

新容器启动时会自动修复已有共享目录的所有权，无需手动修改数据库或配置文件。
