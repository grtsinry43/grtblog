# GrtBlog v2.1.1

- 发布日期（UTC）：2026-07-14
- 对比基线：v2.1.0
- 发布通道：Stable

v2.1.1 重点改善评论阅读、内容编辑和日常升级体验，并补充手记天气与心情信息。此次更新包含评论表结构迁移，升级时会由 Server 自动执行。

## 评论与互动

- 评论改为“根评论分页、回复按楼层平铺”的展示方式，深层回复不会继续缩进，移动端和长对话更易阅读。
- 回复仍会保留精确的目标评论和昵称，点击引用可定位到对应楼层。
- 评论线程新增稳定的根评论和深度信息，最大支持 10 层回复；根评论分页后只加载当前页对应的回复。
- ActivityPub 评论同步适配新的线程结构，服务端、后台协议和前台类型保持一致。

## 内容创作与后台

- 手记可记录天气和心情，并在详情页及实时预览中展示对应的氛围图标。
- 文章标签与手记话题统一使用可搜索、可创建的选择器，减少重复项并改进编辑预览的数据同步。
- 文章、手记和独立页面列表新增按需加载的快捷预览，不再在列表打开时提前请求正文。
- 热门文章提示会读取后台当前配置，正确展示浏览、点赞和评论阈值。
- 编辑器斜杠菜单不再显示仅供服务端生成的联合引用和提及区块。
- 独立页面在内容更新日期与创建日期不同时会显示更新时间。

## 部署与维护

- 一键升级完成后会扫描 `grtblog-server` 和 `grtblog-renderer` 的历史镜像，支持按编号、区间或全选进行批量清理。
- 当前运行镜像会自动排除；健康检查失败时保留全部旧镜像以便回滚，非交互模式也不会自动删除。
- 修复升级过程中已选版本、更新通道和镜像源可能被旧 `.env` 覆盖的问题。
- 调整持续集成配置，修复 CNB 源同步并移除不再使用的 Qodana 工作流。

## 升级须知

推荐重新运行一键安装脚本完成升级：

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/grtsinry43/grtblog/main/deploy/install.sh)
```

国内网络可使用：

```bash
bash <(curl -fsSL https://cnb.cool/grtsinry43/grtblog/-/git/raw/main/deploy/install.sh)
```

- Server 启动时会自动执行 `0066_flatten_comment_threads.sql`，为现有评论补充根评论和深度信息。
- 迁移会校验孤立、循环、跨评论区或超过 10 层的既有评论链；若发现异常会停止迁移，避免静默破坏线程结构。
- 升级健康检查通过后才会询问是否清理历史镜像。如需保留快速回滚能力，可直接回车跳过。
